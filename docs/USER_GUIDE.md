# EyeFocus Insight — 用户手册

> **版本**：v1.0 | **日期**：2026-06-13
> **适用对象**：非技术终端用户

---

## 一、安装

### 1.1 系统要求

| 项目 | 最低要求 | 推荐 |
|------|---------|:----:|
| 操作系统 | Windows 10 | Windows 11 |
| 内存 | 8 GB | 16 GB |
| 摄像头 | 720p @ 30 FPS | 1080p @ 60 FPS |
| Python | 3.10 | 3.12 |
| 硬盘空间 | 500 MB | 1 GB |

### 1.2 安装步骤

**第一步：安装 Python**

从 [python.org](https://www.python.org/downloads/) 下载 Python 3.12，安装时**勾选 "Add Python to PATH"**。

**第二步：下载项目**

```bash
# 打开终端（CMD 或 PowerShell）
git clone https://github.com/cacao1357/EyeFocus-Insight.git
cd EyeFocus-Insight
```

> 如果没有 Git，可点击 GitHub 页面绿色的 "Code" → "Download ZIP"，解压后进入文件夹。

**第三步：创建虚拟环境**

```bash
python -m venv .venv312
.venv312\Scripts\activate
```

> 终端前面出现 `(.venv312)` 表示虚拟环境已激活。

**第四步：安装依赖**

```bash
pip install -r requirements.txt
```

> 安装约需 2-5 分钟。如果下载慢，可尝试清华镜像：
> ```bash
> pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
> ```

## 二、运行程序

### 2.1 启动

在项目目录下运行：

```bash
.venv312\Scripts\python.exe main.py --qt
```

> 如果只是 `python main.py` 可能启动旧版 OpenCV 界面。**推荐加 `--qt` 参数**。

### 2.2 首次使用 — 校准

程序启动后会自动弹出校准对话框，按以下步骤操作：

#### 步骤 1：睁眼基线（5 秒）

1. 校准对话框显示「步骤 1/4 · 睁眼基线」
2. 点击 **「开始」** 按钮
3. **保持自然睁眼**，面向摄像头
4. 右侧倒计时从 5 秒走到 0
5. 系统采集你的正常睁眼 EAR 数据

#### 步骤 2：闭眼检测（3 秒）

1. 显示「步骤 2/4 · 闭眼检测」预告
2. 点击 **「开始」**
3. **轻轻闭上双眼**保持闭合
4. 系统采集闭眼状态的 EAR

#### 步骤 3：头部姿态（4 个方向，每个 2 秒）

会依次进行 4 次，每次都有独立的预告和开始按钮：

| 方向 | 动作 | 持续 |
|:----:|------|:----:|
| 👆 向上 | 抬头看天花板 | 2 秒 |
| 👇 向下 | 低头看地面 | 2 秒 |
| 👈 向左 | 转头看左边 | 2 秒 |
| 👉 向右 | 转头看右边 | 2 秒 |

> 每个方向结束后会显示下一步预告，点击「开始」继续。

#### 步骤 4：眨眼计数（8 秒）

1. 显示「步骤 4/4 · 眨眼计数」预告
2. 点击 **「开始」**
3. **正常眨眼**，心里默数你的眨眼次数
4. 倒计时结束后弹窗输入你数的次数
5. 点击确定

#### 校准完成

页面显示校准结果，包括：
- 睁眼基线 EAR 值
- 头部姿态范围（左右角度、上下角度）
- 可点击 **「进入监测」** 开始使用，或 **「重新校准」** 重来

### 2.3 实时监测

校准完成后进入监测主窗口：

```
┌──────────────────────────────┐
│                              │
│       摄像头实时画面          │
│                              │
├──────────────────────────────┤
│ ⚠ 光照不足（如有）            │
│ ⚠ 人脸丢失（如有）            │
│                              │
│ [😊疲劳]  [--专注时长]  [专注度]  [👁眼睛]  [👤人脸]│
│                  圆环                               │
│             FOCUSED/NORMAL                          │
│             /DISTRACTED                             │
│                              │
│    [暂停] · [校准]          │
└──────────────────────────────┘
```

**卡片说明：**

| 卡片 | 显示内容 | 状态含义 |
|------|---------|---------|
| 疲劳 | 清醒 / 关注 / 疲惫 | 🟢清醒 🟠关注 🔴疲惫 |
| 专注时长 | XXm / XhXm | 累计专注时间 |
| 专注度圆环 | 专注 / 一般 / 分心 | 🟢专注 🟠一般 🔴分心 |
| 眼睛 | 正常 / 闭眼 | 🟢检测到 🔴未检测到 |
| 人脸 | 正常 / 丢失 | 🟢检测到 🔴未检测到 |

**按钮操作：**

| 按钮 | 功能 |
|------|------|
| 暂停 | 暂停/继续实时监测 |
| 校准 | 重新进行校准 |
| 窗口 X | 关闭程序，生成报告 |

**自动行为：**

- **人脸丢失**：超过 10 秒未检测到人脸 → 自动暂停，显示红色警告
- **人脸恢复**：人脸重新出现 → 自动继续监测
- **低光照**：环境光线不足 → 显示黄色警告，自动降低检测阈值

## 三、查看报告

### 3.1 报告位置

关闭程序后，报告自动生成到 `reports/` 目录：

```
EyeFocus Insight/
└── reports/
    ├── {session_id_1}.html     # 第一次会话报告
    └── {session_id_2}.html     # 第二次会话报告
```

### 3.2 报告内容

HTML 报告包含 9 个章节：

| 章节 | 内容 |
|------|------|
| 1. 会话概览 | 时长、数据质量、关键指标 |
| 2. 专注度趋势 | 全时段专注度评分曲线 |
| 3. 疲劳分析 | 疲劳等级分布和时间线 |
| 4. 分心分析 | 分心事件热力图和统计 |
| 5. 数据分析 | PERCLOS、眨眼率等详细指标 |
| 6. 洞察发现 | 变点检测、异常检测结果 |
| 7. 影响因素 | 各因素对专注度的效应量评估 |
| 8. 个性化建议 | 基于数据的针对性改善建议 |
| 9. 数据说明 | 指标含义和使用建议 |

### 3.3 在浏览器中查看

在 `reports/` 目录找到 `.html` 文件，**双击**即可在默认浏览器打开。所有图表和样式已内嵌，无需网络连接。

## 四、常见问题

### 4.1 启动时报错 "摄像头无法打开"

1. 检查摄像头是否被其他程序（如 Zoom、微信视频）占用
2. 其他软件占用 → 关闭占用摄像头的程序后重试
3. 外接摄像头 → 检查 USB 连接
4. 尝试修改摄像头索引：`python main.py --qt --camera 1`

### 4.2 校准总是失败

- 确保环境光线充足
- 确保面部正对摄像头，不要侧脸
- 戴眼镜用户：系统会自动检测，保持眼镜清洁
- 多次失败 → 可尝试 `python main.py`（OpenCV 版校准）

### 4.3 帧率很低

- 确保不是电池模式（插电使用性能更好）
- 关闭其他占用 CPU 的程序
- 降低摄像头分辨率（修改 `config.py`）

### 4.4 报告内容是英文还是中文？

全部内容为**中文**，包括标签、分析结论和个性化建议。

### 4.5 数据会传到云端吗？

不会。所有数据存储在本地 SQLite 数据库中，**零网络请求**，不存储图像帧。

## 五、卸载

### 删除项目

直接删除 `EyeFocus Insight` 文件夹即可，系统无残留：

```bash
rm -rf EyeFocus\ Insight
```

> 如需保留历史数据，先备份 `data/` 目录下的 `.db` 文件和 `reports/` 目录。
